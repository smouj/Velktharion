# Contributing

Thanks for contributing to this project.

## Development flow
1. Fork and create a feature branch.
2. Keep commits atomic and descriptive.
3. Add/adjust tests for behavior changes.
4. Open a PR with context, impact, and rollback notes.

## Quality gates
- lint
- typecheck
- test
- build (when applicable)

## Code standards
- Python 3.11+
- Type hints required for public functions
- Keep modules cohesive and documented
